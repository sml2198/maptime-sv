⚀ GIS: geographic information systems
• ESRI ArcGIS: beloved, powerful & proprietary
• QGIS: open source & scrappy
• misc other: special for web maps, design, etc.
⚁ data layers:
• GIS maps are composed of map layers, drawn in a particular order
• each layer is a dataset that represents a discrete entity (e.g. streets, trees, parks) a map layer defines how a GIS dataset is symbolized and labeled (that is, portrayed) in your map views NOTE: WE NEED A VISUAL HERE
⚂ data types: vector and raster
• a data type is a way of representing data
• all GIS data types consist of latitude and longitude (an x and y coordinate)
• the difference is what data is stored in each point/pixel
vector data:
• is split into three types: polygon, line and point data
• can store many attributes per point or pixel
raster data:
• is also called grid data, and it represents surfaces
• only stores one attribute per point or pixel, e.g. elevation or color ADD LINKS TO WORDS RASTER AND VECTOR TO MAKE THEM STAND OUT
• at the most basic level, think of vector data as a data table, and think of raster data as an image
• today, we are only dealing with vector data
• more info.

⚄ projection systems
• a map is flat (2D). the world is not (?D).
• displaying a 3D object on a 2d surface requires some distortion
• there are many different ways of doing that distortion, and each has different virtues and vices http://www.radicalcartography.net/index.html?projectionref http://www.quadibloc.com/maps/mapint.htm http://geokov.com/education/map-projection.aspx Projections are also classified in terms of their properties. Specifically, on the basis of two very important properties: whether they are conformal, or equal-area, or neither. A conformal projection maintains the shape of small regions, so angles at any point are correct, although sizes will change. An equal-area projection, on the other hand, maintains size at the expense of shape. Maintaining both size and shape, of course, requires a globe.

⚀ navigate ArcMap + make a document
• create a folder and give it a name with no spaces
• go to programs ➔ ArcGIS ➔ ArcMap (v 10.6.1)
• select "blank map"
• save to your folder (a doc in ArcMap is called an .mxd file) make a folder on the desktop (or wherever) and give it a name (make sure there are no spaces in your folder name) we will be using ArcMap today, which is the 2D mapping software that is part of the ESRI ArcGIS suite (ESRI also makes 3d rendering software, called ArcScene and other fun things) first, go to programs -> ArcGIS -> ArcMap (should be version 10.6.1 but not a huge deal if not) select "blank map" (you may need to navigate through "New Maps" -> "My Templates" -> "Blank Map" in the left-hand table of contents) save your file - ArcMap files are saved as an .mxd (Map Exchange Document) which are very portable you can double-click on an .mxd file once it is saved to open it and ArcMap will launch
• the layer panel/TOC is your friend
• note the zoom, pan, full-extent, and select buttons
• toggle between data view and layout view ("View" in the top bar) data view is for working in, and layout view is for getting things print-ready the layer panel, or TOC, displays your layers, which you can turn on and off, drag to reorder, and right-click on to view their properties and attributes 1. the Zoom Tool works exactly as you would expect. 2. the Pan Tool changes the Extent of your Data Frame, without changing the scale. 3. the Full Extent Button zooms you to the full extent of the layer in your Map Document with the largest spatial extent. This can sometimes be problematic if you are working at a local leve, but using one or more layers that are global in extent (for example, many of the network basemap services). 4. the Back/Forward Button works much like its analogous tool in your browser, allowing you to step back through previous changes in Scale/Extent (e.g. if you make a mistake) 5. you can even make a spatial bookmark!
The Data Frame is where your geographic datasets will be visualized Table of Contents is where you will interact with your data most of the time. As you add geographic and non-geographic datasets to your Map Document, they will show up in the Table of Contents as Layers. Much of your work in ArcMap will be initiated by right-clicking on one or more of the Layers in the Table of Contents. Catalog Window – The Catalog Window functions much as Explorer does in Windows. In this window, you can create connections to drives, folders and network resources for use in ArcGIS. Many of the tasks related to data maintenance (creating empty shapefiles/feature classes, new Geodatabases, etc…) are performed using the Catalog Window. This window acts as a ‘lite’ version of the more fully featured ArcCatalog component of the ArcGIS Suite. Toolbars – There are literally dozens of Toolbars in ArcMap. Each of these toolbars is composed of a set of buttons that launch tools that are related to a GIS task of some sort.

quick: what's in a shapefile? thanks, http://jonathansoma.com/tutorials/mapping/parts-of-a-shapefile/
.shp is the geometry, i.e. the points, lines or polygons. basically, it's the shapes!
.prj is the projection system for your shapes
.dbf is the data associated with each shape (e.g. whatever attributes you have stored)
.shx is an index file for the geometry (tl;dr: this speeds things up)
.shp.xml is “geospatial metadata in XML format”. . . yeah, don't worry about that . . .
