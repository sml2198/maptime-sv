⚀ GIS: geographic information systems
• ESRI ArcGIS: beloved, powerful & proprietary
• QGIS: open source & scrappy
• misc other: special for web maps, design, etc.
⚁ data layers:
• GIS maps are composed of map layers, drawn in a particular order
• each layer is a dataset that represents a discrete entity (e.g. streets, trees, parks) a map layer defines how a GIS dataset is symbolized and labeled (that is, portrayed) in your map views
⚂ data types: vector and raster
• a data type is a way of representing data
• all GIS data types consist of latitude and longitude (an x and y coordinate)
• the difference is what data is stored in each point/pixel
vector data:
• is split into three types: polygon, line and point data
• can store many attributes per point or pixel
raster data:
• is also called grid data, and it represents surfaces
• only stores one attribute per point or pixel, e.g. elevation or color
• at the most basic level, think of vector data as a data table, and think of raster data as an image
• today, we are only dealing with vector data
• more info.

• points: a single pixel with a single x,y coordinate
• lines: two points or more connected with lines
• polygons: a bounded area of points connected by lines

⚄ projection systems
• a map is flat (2D). the world is not (?D).
• displaying a 3D object on a 2d surface requires some distortion
• there are many different ways of doing that distortion, and each has different virtues and vices Projections are also classified in terms of their properties. Specifically, on the basis of whether they are conformal, or equal-area, or neither. A conformal projection maintains the shape of small regions, so angles at any point are correct, although sizes will change. An equal-area projection, on the other hand, maintains size at the expense of shape.

quick: what's in a shapefile? thanks, http://jonathansoma.com/tutorials/mapping/parts-of-a-shapefile/
.shp is the geometry, i.e. the points, lines or polygons. basically, it's the shapes!
.prj is the projection system for your shapes
.dbf is the data associated with each shape (e.g. whatever attributes you have stored)
.shx is an index file for the geometry (tl;dr: this speeds things up)
.shp.xml is “geospatial metadata in XML format”. . . yeah, don't worry about that . . .

fortunately, we're not done with projection systems! in ArcMap, datasets can have their own defined projection systems, while the data frame can have a completely different one this is actually a really good thing - it means you can add layers with different systems to the map at once (by using "on the fly" projecting in ArcMap, but this does make your layers render more slowly)
to change the projection system: right-click on Layers → select Properties → select Coordinate System → search for NAD 1983 StatePlane New York → expand the folders until you see NAD 1983 (US Feet) → select the Long Isl one → select Apply
we don't have time to do a deep dive into which projection to use, but generally you want to use a State Plane system for state/local mapping and Albers Equal Area Conic if you're mapping the USA
if you're a stickler, and need to know the difference between a coordinate system and a projection system, read more here
